# 🌊 Ripple 설치 (1분, 설정 0)

> 코드 저장하는 순간, 그 변경이 **누구 코드의 어디를 어떻게 깨는지** 자동으로 알려주는 도구.
> 같은 사무실(같은 와이파이)이면 **그냥 설치만** 하면 자기들끼리 알아서 연결됩니다.

## 1. 파일 받기
공유받은 **`ripple.vsix`** 파일을 다운로드 (바탕화면 등 아무 데나).

## 2. 설치 (VS Code · Cursor 동일)
1. VS Code(또는 Cursor) 열기
2. 왼쪽 **확장(Extensions) 아이콘** 클릭 — 또는 `Cmd+Shift+X` (Windows는 `Ctrl+Shift+X`)
3. 확장 패널 **우측 상단 `···`** → **`Install from VSIX...`**
4. 받은 **`ripple.vsix`** 선택 → 설치

> **Windows**: 첫 실행 때 방화벽이 "네트워크 접근 허용?"을 물으면 **허용** (팀 연결에 필요).

## 3. 끝 — 자동으로 됩니다
- 같은 와이파이의 팀원들과 **자동 연결** (IP 설정 같은 거 안 함)
- 같은 프로젝트(repo)를 열면 **자동으로 같은 팀** (다른 프로젝트와는 분리)
- 화면 왼쪽에 🌊 **Ripple** 아이콘(변경 피드), 아래 상태바에 `Ripple` 연결 표시

## 써보기
- 평소처럼 코딩하다 **저장(`Cmd/Ctrl+S`)**.
- 누가 함수 시그니처·타입·스키마를 바꾸면 → 그걸 쓰는 사람에게
  **"🌊 너의 OOO.ts:42 가 이렇게 깨진다"** 알림이 자동으로 뜸. 클릭하면 그 줄로 점프.

## 혼자 미리 보고 싶으면
`Cmd/Ctrl+Shift+P` → **`Ripple: 데모`** 실행 →
가상 동료가 내 파일이 쓰는 함수를 바꾼 것처럼 시뮬레이트돼서, 알림·변경 피드가 어떻게 뜨는지 바로 확인.

## 설정 (보통 안 건드려도 됨)
- `ripple.userId` — 팀에서 보일 내 이름 (비우면 컴퓨터 사용자명)
- 그 외는 전부 자동.

---
문제가 있으면 (연결 안 됨 등): 명령 팔레트 → `Ripple: 백엔드 재연결`, 또는 출력 패널(View→Output→`Ripple`) 로그 확인.
