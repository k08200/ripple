# 🌊 Ripple

### 라이브 협업 에디터 + AI 변경 영향 분석

**팀이 같은 코드를 라이브로 함께 편집하고, AI가 그 변경이 누구에게 영향을 주는지 대신 읽어 알려주는 협업 도구.**

## ❓ 문제

코드는 **계속** 바뀝니다. 그런데 그 변경을 다른 사람 — 특히 다른 직군 — 은 **늦게, 따로, 단편적으로** 알게 됩니다.

- PR/푸시는 이미 끝난 **스냅샷**이라 "지금 무엇을 하는 중인지"가 보이지 않습니다.
- **프론트엔드**는 백엔드가 어디까지 했는지 몰라 답답하게 기다립니다.
- **백엔드**는 자기 변경이 프론트에 어떤 영향을 주는지 모릅니다.

> Git만 봐서는 절대 "라이브"를 볼 수 없습니다. Git은 커밋된 결과만 보니까요.

---

## 💡 해결: 공존(共存) + 당김

Ripple의 모든 것은 하나의 공식에서 나옵니다.

| 요소 | 역할 | 빠지면 |
|---|---|---|
| **에디터 (공존)** | 같은 코드를 라이브로 보고 함께 편집할 *이유*를 준다 | AI만 있으면 → 볼 화면이 없다 |
| **AI (당김)** | "이 변경, 너에게 영향 간다"며 *주의를 끈다* | 에디터만 있으면 → 또 하나의 죽은 협업툴 |

**둘이 붙어야 작동합니다.** 프론트는 진행 상황을 라이브로 보고, AI가 찔러주니 백엔드도 본다.

---

## ⚙️ 작동 구조

편집을 에디터 안에서 잡기 때문에 **편집 이벤트를 직접 소유**합니다.
Git은 커밋만 보지만, Ripple은 *저장하는 순간*을 봅니다 — 그래서 라이브가 가능합니다.

```
   ┌─ VS Code 익스텐션 ──┐   ← Cursor · Windsurf 자동 포함
   ├─ JetBrains 플러그인 │      (얇은 어댑터:
   ├─ Zed / Neovim ...   ├───   편집을 보내고 / 결과를 그릴 뿐)
   └─────────────────────┘
              │
              ▼
   ┌──────────────────────────────┐
   │      공유 백엔드 (진짜 두뇌)      │
   │   · Yjs (CRDT) 라이브 동기화     │
   │   · AI 변경 영향 분석            │
   └──────────────────────────────┘
```

> **두뇌는 백엔드, 에디터는 얇은 클라이언트.**
> 그래서 에디터를 추가하는 것 = 두뇌를 다시 짜는 게 아니라 *어댑터 하나 더 붙이기*.

---

## 🎯 핵심 차별점

차별점은 **에디터가 아닙니다.** (그건 Cursor · Replit · Codespaces가 이미 합니다.)

> 차별점은 **"AI가 전사(全社) 코드 변경을 사람 대신 읽고, 너에게 영향이 있는지 알려주는 것."**
> 에디터는 그걸 담는 그릇일 뿐입니다.

---

## 🧩 에디터 커버 전략

| 에디터 | 작업량 |
|---|---|
| VS Code + **Cursor** + **Windsurf** + Code OSS 포크들 | **익스텐션 1개로 동시 커버** (모두 같은 확장 API) |
| JetBrains / Zed / Neovim … | 백엔드 두뇌는 그대로, **얇은 어댑터만** 추가 |

익스텐션 하나로 이미 개발자 대다수를 덮고, 나머지는 점진적으로 확장합니다.

## 🛠️ 기술 스택

- **에디터 코어** — [Monaco](https://microsoft.github.io/monaco-editor/) (VS Code 엔진)
- **실시간 협업** — [Yjs](https://yjs.dev/) (CRDT) + `y-monaco`
- **전송** — WebSocket
- **AI 레이어** — Claude Code / OpenAI Codex API (변경 diff → 영향 분석)
- **배포 단위** — VS Code Extension (VSIX, Open VSX)

## 🚫 설계 원칙 (스코프 관리)

- ❌ VS Code를 바닥부터 만들지 않는다 → **Monaco 위에 올린다**
- ❌ 키 입력 단위 실시간에 집착하지 않는다 → **파일 저장 단위면 충분** (반쯤 쓴 코드 분석은 노이즈)
- ❌ 처음부터 전 에디터 · 전사 repo를 노리지 않는다 → **VS Code 패밀리 + 한 팀 2–3개 repo**부터
- ❌ 채팅을 직접 구현하지 않는다 → **AI 대화가 그 자리를 대체한다**
